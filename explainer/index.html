System Design

Full Pipeline Architecture

Six components. One continuous loop. From raw packet to dashboard update in under two seconds.

Step by Step

From packet to blocked IP

Follow a single malicious packet through every layer of the system.

01

Suricata watches every packet

The Pi runs Suricata in network monitoring mode, capturing all traffic on its interface. Every packet is tested against 40,000+ Emerging Threats rules covering port scans, exploits, malware command-and-control beacons, and DoS flood patterns.

alert tcp any any -> $HOME_NET 22 ( msg:"ET SCAN Nmap SSH Version Detection"; flow:to_server; flags:S; ...)
02

tailer.py streams eve.json in real time

A Python process uses Linux inotify to watch Suricata's log file. The instant Suricata writes a new alert line, tailer.py reads it, normalises the numeric severity (1=critical, 2=high, 3=medium, 4=low), and forwards it to the Node.js backend with retry logic on failure.

{"event_type":"alert","src_ip":"91.108.4.55", "alert":{"severity":1,"signature":"ET DOS..."}}
03

The backend enriches and stores the alert

The Node.js server deduplicates identical alerts within a 30-second window, classifies the attack type (RECON, DOS, EXPLOIT, MALWARE), runs a GeoIP lookup, detects slow port scans across a 30-minute window, and writes the enriched record to Supabase.

{ src_ip: "91.108.4.55", country: "RU", attack_type: "DOS", severity: "critical", grouped: false, count: 1 }
04

An LLM explains it in plain English

For high and critical severity alerts, the backend sends the alert metadata to OpenRouter asynchronously. The model returns a structured explanation covering what happened, the business risk, and suggested next steps. This is saved back to the alert row in Supabase.

{ "situation": "A machine sent repeated flood packets...", "exposure": "Could disrupt your internet connection.", "guidance": "The IP has been blocked automatically." }
05

The dashboard updates instantly, from anywhere

The React dashboard subscribes to Supabase Realtime over a WebSocket. New alerts appear in the log, stat cards update, and the threat map places a marker, all without a page refresh, from any device connected to the internet.

06

Auto-block closes the loop

If an IP triggers five or more high or critical alerts within five minutes, the backend calls iptables to drop all further packets from that IP. Manual blocks from the dashboard write to Supabase, which the Pi picks up via Realtime and executes, with no direct network path required between the dashboard and the Pi.

sudo iptables -A INPUT -s 91.108.4.55 -j DROP
